# Contributing to My Attend

We appreciate your interest in helping improve My Attend! Whether you're reporting bugs, suggesting features, or providing feedback, your contributions make our app better for everyone.

## Ways to Contribute

### ğŸ› Reporting Bugs

Found a bug? Help us fix it by providing detailed information.

#### Before Reporting a Bug

1. **Check if it's already reported** - Search existing issues first
2. **Try basic troubleshooting** - Restart the app, check for updates
3. **Reproduce the issue** - Make sure it happens consistently
4. **Gather information** - Screenshots, device details, steps to reproduce

#### How to Report Bugs

**Email Method:**
Send detailed bug reports to [bugs@myattend.com](mailto:bugs@myattend.com)

**In-App Method:**
1. Go to **Profile > Help & Support**
2. Tap **"Report a Bug"**
3. Fill out the bug report form
4. Submit with screenshots if applicable

#### What to Include in Bug Reports

**Required Information:**
- **App version** (found in Profile > About)
- **Device information** (iPhone 12, Samsung Galaxy S21, etc.)
- **Operating system** version (iOS 15.2, Android 12, etc.)
- **Steps to reproduce** the bug
- **Expected behavior** vs. actual behavior
- **Screenshots or videos** if helpful

**Example Bug Report:**
```
Subject: Check-in button not responding on Android

App Version: 1.2.1
Device: Samsung Galaxy S22
OS: Android 13
Organization: ABC Company

Steps to reproduce:
1. Open My Attend app
2. Tap the "Check In" button
3. Button appears pressed but nothing happens
4. No check-in is recorded

Expected: Check-in should be recorded with timestamp
Actual: Button press has no effect

Additional info: This started happening after the latest app update.
Happens every time I try to check in.
```

### ğŸ’¡ Feature Requests

Have an idea for a new feature? We'd love to hear it!

#### How to Suggest Features

**Email Method:**
Send feature suggestions to [feedback@myattend.com](mailto:feedback@myattend.com)

**In-App Method:**
1. Go to **Profile > Help & Support**
2. Tap **"Suggest a Feature"**
3. Describe your idea in detail
4. Submit your suggestion

#### What Makes a Good Feature Request

**Include These Details:**
- **Clear description** of the proposed feature
- **Use case** - Why would this be helpful?
- **Target users** - Who would benefit from this feature?
- **Priority level** - How important is this to you?
- **Alternative solutions** - What do you currently do instead?

**Example Feature Request:**
```
Subject: Bulk check-in for team events

Description: 
Add ability for managers to check in multiple employees at once 
for team events, meetings, or training sessions.

Use Case:
When our team has off-site training, it's tedious for everyone 
to individually check in. A bulk check-in would save time and 
ensure everyone's attendance is recorded.

Target Users:
Managers and team leads who organize group activities

Priority: Medium
Currently we manually adjust attendance records after events.

Suggested Implementation:
- "Bulk Check-in" option in admin panel
- Select multiple employees from list
- Set common check-in time
- Add notes about the event
```

## Beta Testing Program

Get early access to new features and help shape the future of My Attend.

### Beta Program Benefits

- **Early access** to new features (2-4 weeks before general release)
- **Direct feedback channel** to our development team
- **Influence product direction** with your input
- **Priority support** for beta-related issues
- **Exclusive updates** on development progress

### How to Join the Beta Program

1. **Email [beta@myattend.com](mailto:beta@myattend.com)** with "Beta Program" in the subject
2. **Include your information**:
   - Organization name
   - Your role (admin, manager, employee)
   - Current app version you're using
   - Device type and OS version
3. **We'll respond within 48 hours** with beta access instructions
4. **Install the beta version** following our instructions
5. **Start testing** and providing feedback

### Beta Testing Guidelines

**What We Expect:**
- **Regular usage** of beta features in your daily workflow
- **Timely feedback** on bugs and usability issues
- **Participation** in beta surveys and feedback sessions
- **Patience** with beta bugs and incomplete features

**What You Can Expect:**
- **Weekly beta updates** with new features and fixes
- **Direct communication** with our development team
- **Advance notice** of major changes or new features
- **Recognition** in release notes for significant contributions
